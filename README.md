# MediKeep+ | Smart Family Health & Pharmacy Tracker
MediKeep+ is a high-level healthcare management tool engineered to simplify how families track their wellbeing. I built this focusing on one core mission: clarity. In a household, keeping track of everyone’s medical routine is often chaotic, so I engineered a seamless interface that bridges the gap between digital inventory and daily family safety.

### Key Features
* **Family Health Synchronization:** A centralized monitoring dashboard that tracks the "Taken/Pending" status of medical routines for the entire family in real-time.
* **Smart Inventory Engine:** I built a dynamic stock management system that monitors tablet counts and provides instant visual alerts when supplies hit critical levels.
* **Instant Navigation Logic:** A custom state-driven architecture that allows users to switch between the Home, Dashboard, and Pharmacy Box without any loading lag.
* **Healthcare-Grade UI/UX:** Designed with a clean "Sky Blue & Ivory" aesthetic to ensure maximum trust, readability, and ease of use for all age groups.
* **Responsive Data Grids:** Implemented CSS Grid-based layouts that automatically optimize healthcare data for pixel-perfect clarity on both mobile and desktop.

### Why I Built This
Most health apps are either too cluttered or locked behind subscriptions. I wanted to see how I could use React functional components and State management to build something life-improving yet incredibly simple. I handled the UI architecture and the inventory logic myself to ensure the app stays lightweight and responds instantly. It’s about tech serving a human purpose keeping families healthy and organized.

### Tech Stack
* **React.js:** For the core component architecture and dynamic state handling.
* **Modern CSS3:** For custom grid systems, interactive cards, and responsive layouts.
* **JavaScript (ES6+):** For managing complex family data arrays and inventory logic.

### How to Use
* Clone the repository.
* Run **npm install** to get all dependencies.
* Run **npm start** to launch the digital pharmacy.
* The app will open at **localhost:3000**, ready for use.
